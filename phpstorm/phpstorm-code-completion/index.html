PhpStorm 的 Auto Complete 再進化

讓你打更少的字加快開發速度
Contents
  1. Version

  3. 再進化
  4. params

你是否常常因為 method 參數要填什麼猶豫不決呢? 可能要去翻 API 文件? 要去 trace 其 source code? 知道參數填什麼後，還要去手動輸入該帶入什麼變數? 在 PhpStorm 2016.2，這些全部都可以自動化，讓你用完全不用打任何字，只要用選的就可以了。

Version

PhpStorm 2016.2
PHP 7.0.0
Laravel 5.2.22

Auto Complete

PhpStorm 2016.2 之前的 auto complete，讓你選擇可以選擇 method，並提供參數資訊，老實說這樣已經很好用了。

再進化

PhpStorm 2016.2 對於參數的 auto complete 做了更強的支援，不過預設並沒有啟動。

PhpStorm -> Preferences -> Editor -> General -> Smart Keys

  • 將 Enable smart function parameters completion 打勾。

以後選擇 method 後，PhpStorm 還會幫你將原 method 的參數名稱複製過來，讓你可以更清楚要填什麼參數。

除此之外，若你使用的 PHP 7 的 type hint，或使用 PHPDoc 的 @param 描述參數型別，PhpStorm 會幫你根據目前 scope 列出相同型別的變數讓你挑選，你也可以輸入變數名稱的前幾個字母來 filter。

通常要傳入的變數就在 method 附近，PhpStorm 也會很聰明的將這些變數優先權放在最前面讓你選，大部分狀況下抓到都是對的變數，只要按 enter 即可。

按 enter 選擇了變數後，PhpStorm 會自動跳到下一個參數，也讓你用選的。

params

若你不想自動帶入參數，也可以在參數列下輸入 params，PhpStorm 會自動幫你將原 method 的參數名稱複製進來，之後一樣可以用選的。1 1此圖為 PhpStorm Code Completion in PhpStorm 2016.2 所提供的 demo。

Conclusion

  • 要善用工具改善開發速度，將時間花在真正需要的地方，PhpStorm 2016.2 的 auto complete 讓我們可以打更少的字， 還可以避免 typo。
