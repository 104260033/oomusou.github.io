如何使PhpStorm不要對變數做Spell Check?

讓PhpStorm的每個Warning都有意義
Contents
  1. Version
  2. Problem
  3. Solution
  4. Conclusion

一般我們取變數名稱時，可能會使用英文縮寫，但此時PhpStorm會很盡責的幫我們做Spell Check，若變數的英文拼字錯誤，會在變數下方加上波浪底線，由於PhpStorm有很強悍的warning功能，每個warning都值得我們特別注意，但若因為Spell Check的warning而造成大家神經緊張，那就太不值得了，所以我個人是將此功能關閉。

Version

PhpStorm 10.0.1

Problem

如下圖所示，PhpStorm對$fillable產生warning，理由是Spell Check認為fillable拼字有問題 XD。

Solution

PhpStorm -> Preferences -> Editor -> Inspection
將Spelling與Type都不要打勾。

之後就再也不會在$fillable變數產生warning了。

Conclusion

  • 由於PHP並不是強型別語言，無法由compiler幫我們檢查語法，只有執行才知道是否錯誤，所以任何PhpStorm的warning對我們都非常重要，除非確定某個warning對我們沒有影響，否則一定要想辦法將warning清除。
