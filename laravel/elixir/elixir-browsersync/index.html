由於HTML/CSS技術越來越複雜，如Dreamweaver那種所見即所得的工具已經很難全面支援HTML/CSS，所以前端開發取而代之的是使用如Sublime Text之類的文字編輯器，然後直接在瀏覽器預覽，此時如 Browsersync 之類的工具就非常方便，讓我們只要直接存檔就可以立即在瀏覽器看結果。Laravel Elixir 3.3 直接支援 Browsersync，讓前端開發更有效率。

Version

OS X 10.11
Laravel 5.1.19
Homestead 0.3.0
Laravel Elixir 3.3
Node.js 0.12.7
npm 2.11.3
gulp 3.9.0

Prerequisite

  • 在Mac已經成功安裝好Homestead。
  • 在Mac已經成功安裝Node.js、nvm與gulp。

安裝Laravel Elixir

Gulp可以幫我們做一些前端的自動化工作，如自動編譯Less與Saas，Babel自動編譯JavaScript ES6，自動執行PHPUnit…等。
Laravel Elixir是Laravel對gulp的再次封裝，讓我們可以更簡單地在Laravel使用gulp。1 1詳細請參考Laravel Elixir
在使用Composer建立Laravel專案時，會自動安裝後端的composer.json內的PHP package，但卻沒有自動安裝前端的package.json內的Node package，需自行手動安裝。

新Laravel 5.1專案

使用composer建立Laravel專案後，在專案目錄下安裝Laravel Elixir。

既有Laravel 5.1專案

因為Laravel Elixir 3.3才支援browsersync，所以必須先更新到3.3版。2 2雖然也可以在Homestead內執行npm install或npm update，但由於npm很耗CPU，建議直接在Mac執行npm，但前提是必須先在Mac安裝好npm。

測試Laravel Elixir

若能看到以下畫面，表示Laravel Elixer已經正常執行，順利編譯Saas。

修改gulpfile.js

16行

增加.browserSync();，表示gulp在執行時會自動啟動browsersync。

啟動gulp watch

gulp watch會幫我們監視，若有任何檔案被修改，就自動執行browsersync更新瀏覽器。
當我們使用Homestead時，以前都是在瀏覽器打http://homestead.app，有了browsersync之後會稍微不一樣，我們必須改打http://localhost:3000，透過proxy機制導到http://homestead.app。3 3在此是將gulp執行在Mac，所以必須事先使用npm install -g gulp將gulp安裝在Mac上，當然也可以在Homestead內執行gulp watch，不過那還必須透過vagrant-notify通知Mac的browsersync，速度會慢很多，所以建議直接在Mac執行gulp watch。

測試Browsersync

在網址列打http://localhost:3000，browsersync會自動透過proxy導到http://homestead.app。

在網址列打http://localhost:3001，可進入browsersync的設定頁面。。

Conclusion

  • 有了 Laravel Elixir + browsersync ，以後我們在修改blade時，就不用再一直手動refresh看結果了，若搭配雙螢幕，只要blade一存檔，browsersync就會自動更新瀏覽器，只要頭轉45度角就可以立即看到結果，非常方便。
  • 由於browsersync與gulp是執行在Mac，而不是執行在Homestead，所以沒有速度上的問題。
  • 這種方式在Windows也可以使用，不侷限在Mac，只要在Windows也裝好npm與gulp即可。
