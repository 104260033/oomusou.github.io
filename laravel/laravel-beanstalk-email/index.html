傳統在發送 email 時，必須以同步方式等待 email 寄送成功，才將控制權交還給使用者，須等待 1 到 2 秒，若使用非同步方式發送 email，則不需等待任何時間。

Motivation

Laravel 支援了幾種常見的 Queue server，如 Beanstalk, Amazon SQS, IronMQ, Redis，其中 Homestead 已經內建 Beanstalk 與 Redis，可直接使用，不需安裝，Laravel 5.1 甚至支援了以資料庫當成 Queue server。不過一般來說，資料庫的負擔已經很重，且資料庫的速度還是比 Queue server 慢，所以建議直接使用 Queue server。

Version

PHP 5.6
Laravel 5.1
Homestead 0.2.7

實際案例

Queue server 最典型的應用就是寄 email，儘管已經使用了 Ajax post，但因為使用同步的方式寄 email，需要確認 email 成功寄出後，才會將控制權交給 JavaScript 的 Ajax post 的 success，所以使用者一樣會等待 1 到 2 秒鐘，若能搭配 Beanstalk server 使用非同步的方式寄 email，使用者將不會有任何等待時間。

安裝 Beanstalk Package

修改 .env

QUEUE_DRIVER 預設是 sync，也就是同步，這樣不會使用到任何 Queue server，因為我們要使用 Beanstalk server，所以改成 beanstalkd。

改用 Mail::queue()

使用 Mail::raw() 或 Mail::send() 都只能使用同步方式寄送 email，只有使用 Mail::queue()，它才會自動使用你所設定的 Queue server 來寄信。

自動執行 queue:listen

Queue server 的機制除了不斷的由 Mail:quene() 塞 job 進 queue 外，還要有一個 listener 不斷執行 queue 內的 job，當然我們可以在 terminal 下 php artisan queue:listen，不過這只適合在開發階段，在正式上線主機我們必須另外開一個 process 執行 php artisan queue:listen，這就必須靠 supervisor，在 Homestead 已經內建 supervisor，可直接使用。

  • directory ：專案在 Homestead 的實際目錄。

Conclusion

  • 非同步機制能大大改善使用者經驗，對於不需要即時的應用，如轉檔縮圖，長時間演算法運算，可以多多使用 Beanstalk queue server。
