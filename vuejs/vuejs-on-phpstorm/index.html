我是一個很依賴auto complete才能寫程式的人，一來我記憶力很差，我記不了很多API，二來我很容易typo，所以只能依賴auto complete用選的，所以寫Vue.js的第一件事情，就是在PhpStorm先把auto complete環境弄好。

Version

Vue.js 1.0.10
PhpStorm 10.0.1

Vue.js on PhpStorm

Vue.js分兩部分，一部分是directive，一部分是API，所以我們接下來要做的就是讓PhpStorm的auto complete能自動顯示directive與API。

Directive

下載Vue.js Plugin

到Vue.js Plugin下載，檔案為vuejs-plugin.jar，已經支援Vue.js 1.0新directive，如v-for。

安裝Vue.js Plugin

PhpStorm -> Preferences -> Plugin -> Install plugin from disk…

載入剛剛下載的vuejs-plugin.jar。

安裝完會要求重新啟動PhpStorm。

自動顯示Directive

重新啟動後，只要輸入v，就會顯示Vue.js專屬的directive了。

API

JavaScript與PHP一樣，所有的auto complete都必須亦賴註解完成，PHP靠的是PHPDoc，而JavaScript靠的是JSDoc。

但若使用的是min版本的CDN link呢？這一定是沒有註解的版本。

PhpStorm提供了External Libraries機制，會自動從CDN下載，並加上JSDoc，因而可以在PhpStorm使用auto complete。

下載External Libraries

CDN反白，表示PhpStorm在本機並沒有含JSDoc版本的library，因此無法使用auto complete。

滑鼠指標放在反白上，按⌥ + ↩，顯示Download library。

PhpStorm會自動下載，並顯示訊息。

確認安裝成功

PhpStorm -> Preferences -> Languages & Frameworks -> JavaScript -> Libraries

會出現vue，表示Vue.js的external library已經安裝成功。

CDN也不再反白了。

Conclusion

  • PhpStorm包含WebStorm，所以也可以拿來開發JavaScript，並對JavaScript加以測試、重構與偵錯，在PhpStorm打造好Vue.js的開發環境後，就可以開心的學習Vue.js了。
